Transitioning from Google Weather API to MSN Weather

08 December 2012

Since Google killed its weather API without a warning over the summer, I have been looking for a new provider for my Weather iOS application. Most of the weather data providers I found were either paid, allowed limited request calls, or had restrictive licenses.
Recently, I came across the MSN Weather API that is quite similar to Google Weather. The feed is accessible from

http://weather.service.msn.com/data.aspx?weadegreetype=f&weasearchstr=94102

In the URL, weadegreetype can be c for celcius or f for fahrenheit, and weasearchstr can either be ZIP code, city, state name or even latitude, longitude.
It provides an XML feed that includes the current and future conditions as a code and text descriptions. This question over at StackOverflow identified various skycodes and their corresponding condition, and time.
If you need weather condition images, you can use http://local.msn.com/worldweather.aspx to see how the skycodes correspond to the weather images. For example, 26 (cloudy) skycode correspond to 26.gif which is the cloudy weather condition image, where the imagerelativeurl can be found in the feed.
